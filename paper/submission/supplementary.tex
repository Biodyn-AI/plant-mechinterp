\section*{Supplementary Methods}

\subsection*{Synthetic sequence generation}

Sequences were generated to reflect key biological properties of each genomic region type:

\begin{itemize}
    \item \textbf{Promoters}: AT-rich background (60\% AT), with TATA box (TATAAA) inserted at positions 20--35, CAAT box at positions 60--80, and optional GC box (GGGCGG) at positions 100--120.
    \item \textbf{Exons}: Balanced nucleotide composition (25\% each), with in-frame stop codons (TAA, TAG, TGA) replaced by random non-stop codons, and optional start codon (ATG).
    \item \textbf{Introns}: AT-rich background (65\% AT), with canonical 5' splice site (GT) at position 0, 3' splice site (AG) at the end, and an A-rich branch point region at the sequence midpoint.
    \item \textbf{Intergenic}: AT-rich background (60\% AT) with optional short tandem repeats (AT, TA, CA, TG) inserted at random positions.
\end{itemize}

\subsection*{Cross-species sequence generation}

Species-specific sequences were generated by sampling nucleotides according to species-appropriate GC content: 36\% for \textit{A.\ thaliana}, 44\% for \textit{O.\ sativa}, and 47\% for \textit{Z.\ mays}. These values reflect documented genome-wide GC content for each species.

\subsection*{Overcomplete SAE architecture details}

The overcomplete sparse autoencoder uses the following architecture:
\begin{itemize}
    \item Encoder: Linear(768, 3072) + ReLU
    \item Decoder: Linear(3072, 768)
    \item Loss: MSE reconstruction + $\lambda \cdot \ell_1$ penalty on encoder activations ($\lambda = 5 \times 10^{-4}$)
    \item Optimizer: Adam with learning rate $10^{-3}$
    \item Input preprocessing: Per-feature standardization (zero mean, unit variance)
    \item Training: 200 epochs, batch size 64
\end{itemize}

This follows the overcomplete dictionary learning paradigm of \citet{bricken2023monosemanticity}, where the dictionary size (3,072) exceeds the input dimension (768) by a factor of 4$\times$, allowing features to be represented as sparse combinations of dictionary elements rather than compressed encodings.
